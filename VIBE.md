# VIBE

This repository uses a markdown-only workflow.

## Read order

1. `AGENTS.md`
2. `.vibe/STATE.md`
3. `.vibe/PLAN.md`
4. `.vibe/HISTORY.md` (optional)

## Core files

- `.vibe/PLAN.md`: checkpoint backlog
- `.vibe/STATE.md`: active checkpoint + status
- `.vibe/HISTORY.md`: archived progress notes
- `.vibe/CONTEXT.md`: short handoff context

## Loop

1. Pick checkpoint
2. Implement small scoped change
3. Review against acceptance
4. Mark done or log issue
5. Continue
